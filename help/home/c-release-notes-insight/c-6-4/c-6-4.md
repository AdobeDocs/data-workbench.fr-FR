# Notes de mise à jour Data Workbench 6.4{#data-workbench-release-notes}

Les notes de mise à jour de Data Workbench 6.4 comprennent les nouvelles fonctionnalités, les exigences de mise à niveau, les bogues corrigés et les problèmes connus.

Pour vue les fonctionnalités et correctifs antérieurs des versions antérieures, consultez l’archive [des notes de](https://docs.adobe.com/content/help/fr-FR/data-workbench/using/release-notes/release-notes.translate.html)mise à jour.

## Atlas du périphérique avec mémoire cache{#device-atlas-with-in-memory-cache}

Le fichier DeviceAtlas.bundle utilise désormais un cache en mémoire pour améliorer considérablement les performances des recherches.

Par défaut, Device Atlas met en cache jusqu’à 100 000 agents utilisateur et leurs propriétés. Le cache LRU est entièrement autonome dans DeviceAtlas.bundle, de sorte que toute version du serveur capable d&#39;utiliser un fichier d&#39;assemblage bénéficiera automatiquement des performances accrues dès que le nouveau fichier sera chargé.

**Modification du fichier DeviceAtlas.cfg**

La taille maximale du cache LRU peut être configurée en modifiant le paramètre Limite des éléments mis en cache dans le fichier DeviceAtlas.cfg.

A compter de la version 6.4 du Data Workbench, vous pouvez remplacer la limite des éléments mis en cache par défaut de &quot;100000&quot; en modifiant sa valeur et en enregistrant le fichier. La valeur par défaut a été sélectionnée pour répondre à la plupart des besoins.

```
component = DeviceAtlasComponent: 
  DeviceAtlas Bundle File = string: Lookups\\DeviceAtlas\\DeviceAtlas.bundle 
  Unsynchronized Bundle Extraction Path = string: Temp\\DeviceAtlas\\ 
  
<b> Cached Elements Limit = unsigned int: 100000</b> 
```

## Conditions requises pour la mise à niveau {#section-84f8f38e9c424d7788c79146fce5a6fe}

Suivez ces exigences et recommandations lors de la mise à niveau vers Data Workbench 6.4.

>[!IMPORTANT]
>
>Il est recommandé d’utiliser les fichiers de configuration par défaut nouvellement installés et de les personnaliser, plutôt que de déplacer des fichiers d’une installation précédente, à l’exception des suivants :

* **ajouter** les processus ****** exclus pour la protection des points de terminaison *MS System Center sur les serveurs* Windows 2012 pour les exécutables suivants :

   * **`InsightServer64.exe`**
   * **`ReportServer.exe`**
   * **`ExportIntegration.exe`**

   Cela permettra des droits de liste blanche pour ces exécutables d&#39;interface.

* **Mettez à jour le *`Trust_ca_cert.pem`*certificat sur les serveurs**.

   * Le dossier *Attribution* a été renommé ***Attribution - Premium*** (situé dans l’installation par défaut à *Profils* \ *Attribution - Premium).*

   * **`Decoding Instructions.cfg`**

* Grâce à cette réorganisation, vous souhaiterez supprimer les anciens dossiers *Attribution* et *Premium* de votre installation serveur.

## Mises à jour système {#section-8e6b9dcaf333453bb2079b151bbbce93}

Ces fonctionnalités ont été renommées, supprimées, ou les fichiers ou dossiers d’installation ont été restructurés dans cette version :

* Le dossier **Base.zip** n’est plus inclus dans le package de mise à jour de version.
* The **DeviceAtlas.bundle** file now uses an in-memory cache to improve the performance of lookups.
* Dans le fichier **Log Processing.cfg** , le paramètre Taille *du* bloc sous Sources *du* journal a été supprimé.
* Dans le fichier **Disk Files.cfg** , le paramètre *Detect Disk Corruption* a été supprimé aux emplacements suivants :

`\server\components\disk files.cfg`

`\server\components for processing servers\disk files.cfg`

* Nouvelles descriptions de service pour les Services ** Adobe Analytics Premium et pour les Services *de rapports* Adobe Analytics Premium dans les propriétés exécutables.
* La fonction d’exportation *de Profil de marketing* Principal du tableau détaillé a été renommée Exportation ***de*** Profils et Audiences.
* La fonction Exportation *de* test et de Cible du tableau détaillé a été renommée Exportation ****** Adobe Target.

## Corrections de bogues {#section-bcdd732af53249b893d30bb42eeb14fc}

Les corrections suivantes ont été apportées au Data Workbench 6.4 (depuis la sortie de Data Workbench 6.31).

* Le score de propension n’était pas réinitialisé lors de la réexécution de différentes entrées dans le même espace de travail. Il est désormais réinitialisé correctement.
* Aucune dimension dénombrable n&#39;est disponible lors de la première ouverture de la matrice de corrélation.
* L’exportation des segments de Cible échouait car le champ mboxPC manquait. Ce problème est maintenant corrigé.
* Format correct de la demande d’ID. L’utilisation de l’identification    mbox3rdpartyId identification instead of default PCIDs caused Adobe Target to reject requests generated via the Target/Data Workbench integration (using the **`ExportIntegration.exe`**). Le format de cette demande d’ID est désormais correct. De plus, le débit fonctionne correctement.
* La fuite de mémoire du serveur de rapports lors de l’exportation vers Excel a été corrigée.

## Problèmes connus {#section-d0cd72e82d1444a2b069dfcda5d7692a}

Les restrictions connues du Data Workbench 6.4 sont les suivantes.

* **`ExportIntegration.exe`**   nécessite des arguments de ligne de commande en anglais. The output file name should be named in English for *Adobe Target Export*, *Profiles and Audiences Export*, and *Customer Record Service Export*.
* Dans l’exportation *des* Profils et des Audiences, la saisie de caractères non autorisés (`[CR]` ou `[TAB]`) sous forme de noms de colonne génère des journaux incorrects, ce qui entraîne une mauvaise exportation des données.
* Dans les versions chinoise et japonaise, un problème de codage de caractères Unicode peut se produire dans le navigateur de chemins.
